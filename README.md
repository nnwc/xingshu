# 星枢 · XingShu

<div align="center">

面向中文用户的 **脚本执行、任务调度与运维控制台**

**当前版本：v1.6.0**

适用于：
**个人服务器 / VPS / 家庭实验室 / 自托管工具集 / 自动化运维场景**

</div>

---

## 项目定位

星枢（XingShu）是一个基于 **Rust + React** 构建的可视化任务管理面板，用来统一管理：

- 定时任务
- 手动任务
- 开机任务
- 脚本文件
- 环境变量
- 任务日志
- 通知推送
- 订阅同步
- 系统配置

如果你已经厌倦了：

- 手写 `crontab`
- 到处散落的 shell / Python / JS 脚本
- 环境变量和依赖没人整理
- 日志查找困难
- 出问题时没有统一入口

那星枢就是把这些能力集中到一个中文界面里，方便长期维护。

## v1.6.0 这一版重点

这一版的重点不是堆功能，而是把“日常真正会用到的体验”做顺：

### 1. 任务管理更适合高频操作
- 支持任务多选与批量操作
- 移动端和桌面端布局都做了适配
- 任务编辑页对多账号运行、通知、执行流程配置更清晰

### 2. 环境变量页体验更完整
- 列顺序和操作区更适合中文使用习惯
- 标签、状态、创建时间、操作等信息更容易扫读
- 页面布局更紧凑，避免横向滚动影响操作

### 3. 暗黑模式整体统一
- 统一了全局文字层级
- 统一了暗黑模式背景基底
- 统一了卡片、表格、输入框、按钮等组件风格

现在暗黑模式不再像“默认组件拼起来”，而更像一套完整界面。

### 4. 多账号运行能力更实用
- 增强了账号拆分规则
- 支持更灵活的拆分方式
- 适合顺序跑号、并发跑号、批量轮询这类场景

### 5. MicroWARP 集成更完整
- 支持系统级维护 MicroWARP 配置
- 任务级可选启用
- 支持运行前切换 IP
- 可以在日志中记录切换过程与出口信息

## 核心功能

### 任务管理
支持三类任务：

- **定时任务**
- **手动任务**
- **开机任务**

支持配置：

- 执行命令
- 工作目录
- 前置命令 / 后置命令
- 分组
- 启用 / 禁用
- 超时控制
- 并发策略
- 多账号运行模式（single / sequential / concurrent）
- 执行完成后的通知推送

---

### 脚本管理
适合集中管理你服务器上的脚本文件：

- 浏览脚本目录
- 新建 / 编辑 / 上传脚本
- 直接运行脚本
- 调试运行
- 查看执行输出
- 区分源脚本与输出目录

适合维护 shell、Python、Node.js 等自动化脚本。

---

### 环境变量管理
星枢提供独立的环境变量面板，用来统一管理任务依赖的变量：

- 新增 / 编辑 / 删除变量
- 启用 / 禁用变量
- 标签与备注管理
- 与任务和脚本共享使用

相比手改 `.env` 文件，更适合长期维护和多任务共用。

---

### 日志与通知
支持统一管理执行结果与推送行为：

- 任务执行日志
- 系统日志
- 登录日志
- 日志保留策略
- 通知模板
- 多种推送渠道

日志保留支持多种策略：

- 按天数清理
- 按全局总数清理
- 按每个任务独立数量清理

通知内容支持：

- `full`：发送完整日志
- `summary`：发送提炼摘要

---

### 订阅同步与导入
适合维护脚本仓库同步场景：

- 订阅远程仓库
- 自动同步脚本
- 自动扫描并导入 Loader 脚本
- 自动分组与命名

适用于你把常用脚本集中维护在 GitHub / GitLab / 其它仓库的场景。

---

### MicroWARP 支持
如果你有代理切换或出口 IP 管理需求，这部分会比较有用：

- 系统级配置代理参数
- 支持代理地址、切换方式、容器名等配置
- 任务运行前可触发切换
- 记录切换前后的过程和结果

适合和多账号任务、轮询任务结合使用。

## 快速开始

### Docker（推荐）

```bash
docker run -d \
  --name xingshu \
  --restart unless-stopped \
  -p 3000:3000 \
  -v $(pwd)/data:/app/data \
  -e TZ=Asia/Shanghai \
  ghcr.io/nnwc/xingshu:latest
```

访问：

```text
http://你的服务器IP:3000
```

---

### Docker Compose

```yaml
services:
  xingshu:
    image: ghcr.io/nnwc/xingshu:latest
    container_name: xingshu
    ports:
      - "3000:3000"
    volumes:
      - ./data:/app/data
    environment:
      - DATABASE_URL=sqlite:///app/data/db/xingshu.db
      - RUST_LOG=info
      - TZ=Asia/Shanghai
    restart: unless-stopped
```

启动：

```bash
docker compose up -d
```

## 运行与数据

默认情况下，星枢会把数据保存在挂载目录中。建议至少持久化以下内容：

- 数据库
- 日志
- 脚本目录
- 配置文件

如果你长期使用，建议配合：

- 自动备份
- WebDAV 远程备份
- 定期检查日志保留策略
